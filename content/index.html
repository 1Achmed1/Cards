Welcome to Cards 1.3!

Welcome to Cards version 1.3 "Cotton Candy"! In this version I've added

  • New content loading system. Allows for mobile and desktop versions to be loaded from the same file, simplifying adding new pages.
  • Added toasts! Toasts can be dismissed and optionally hidden until a button is pressed.
  • Added sample login page and necessary elements to make it look good.
  • Added inset buttons. They can be seen on the login page linked in the above listing.
  • Added colored cards. Automatically selects text color according to Color Guidelines.
  • Added toggling functionality to the nav drawer.
  • Added new animation where cards slide up when the page is loaded.
  • Added variables.js, contains all variables used for jQuery, making content editing a little easier.
  • Added side-by-side cards. See below for details.
  • Added restricted-width container. Container can now be restricted to 50% width, with a min-width of 750 pixels. See the Vertical Navigation page.
  • Added a fancy new setup page for editing site info. // This is WIP and doesn't currently do anything on the site.
  • Reworked padding on the left and right of the header links.
  • Reworked page titling system, works with new Content Loader. Inludes fallback if there's no specified title.
  • Removed line under card title by default. Can still be added with the lower-line class.

The source and the download are both available on the GitHub project page. At the moment Firefox works best with Cards, largely due to how it handles buttons. I'd recommend using it instead of Chrome for the time being.

Section Header.

This is a normal section with a header. The header can be used to display header text like a title.

Example with the whole section:

<div class="card" titled>
<div class="title">
  <span>Section Header.</span>
</div>
<div class="content">
  <p>This is a normal section with a header. The header can be used to display header text like a title.</p>
</div>
</div>
Colored Card

Colored cards deprecate shadowed cards in version 1.3. Shadowed cards, or those where the title has a shadow below it, will be removed in Cards 1.4, and colored cards will replace them.

Example:

<div class="card" titled color="teal">
  <div class="title" teal>
    <span>Colored Card.</span>
  </div>
</div>
Blue-Grey Card

This is just a sample blue-grey colored card. Blue-grey goes well with a teal accent color.

Side-By-Side Cards

In Cards 1.3, you can now place cards side by side, with up to 6 cards horizontally. Side by side cards require Flex element browser support. Browsers that support Flex include Chrome, Firefox, Edge, and Safari 9.

Left Card

Side by side cards are kept in a div with a class of side-by-side-cards, which is a flex element.

Right Card

The Flex position of each side by side card is defined by which child it is. Since this card is the second child of the flex element, it'll be flex: 2;.

Titling Pages

Cards 1.3 introduces a new page loading system. This change means that there's one central index page that dynamically changes. As a result, there needed to be a new system to give page titles. This can be done with a div element.

<div class="meta-title" data-title="Page Title Here!" />

The new system uses jQuery to look for a div with the meta-title class, and then uses the element's data-title attribute to title the page. A default can be set on the core index, but you should make it a generic "Loading..." title, indicating to the user that the website is loading.

Colored Section Header.

Coloring the title is as easy as adding the colored attribute to the title div.

Example:

<div class="title" colored>
<span>Colored Section Header.</span>
</div>

The colored section header automatically takes the @header-color from base.less. Less can be used to make changing things a lot easier, however there's no reason why you cannot change it directly in base.css.

Non-hovering Card

You can prevent a card from using the hover effect with the hover attribute. Cards 1.4 will change custom attributes to data- ones, meaning in Cards 1.4 instead of hover="false", it will be data-hover="false"

Example:

<div class="card" hover="false">

The hover="false" attribute can be used with titled div's, aswell.

The non-hovering card is used for cards that wouldn't natually hover, or that have content on them that would also hover. Examples include the nav drawer, heading card, and login card.

Toasts

Toasts are closed, or off screen, by default. To have a toast opened when the page loads, add the opened class to the corresponding toast div element. An example can be seen on the sample login page. Anything that will show a toast must have the showToast class and a data-toast-toggle value the same as the id of the targeted toast. The toast dismissal system is reliable but the parent of the span with the dismissToast class must be the toast div.
